Getting Ready

In your first programming assignment, you'll add on to the app model that we started building in class. Please download the starter project from the following link to get started:

Downlink Homework 1 Materials (link will be active start of week 1)

This starter project starts where the lecture project left off, so you've already got that code in place.

Don't forget to change the Bundle Identifier (as described in the syllabus), and then rename your project to LASTNAME_Homework1. You can do this by:

  • Select the project file (the top level file) in project navigator and rename the project
  • Xcode will prompt to rename targets and other metafiles as well - select all of them
  • Change the bundle identifier (as described in the syllabus)
  • Change the scheme name (Select scheme drop down, select "Manage Schemes" and rename the scheme from the list of schemes)
  • Rename the top-level directory (the one that contains the .xcodeproj file)

Assignment Requirements

To build on the project, you should do the following:

  • Use a Swift Playground to test out some code before putting it in your project:
    • In Playgrounds/HW1ConfirmationPlayground.playground file, do the following:
    • Copy over the structs that we went over in the lecture
    • Expand the "carrot" next to the Playground name in the file inspector, and copy the JSON files in the resources folder into the Playground's resource folder
    • Make the Student struct, which should have the following properties:
      • The name, address, email, jhed, imageName as Strings.
      • A property for the program to which the student belongs (like "Computer Science" or "Engineering"), but NOT as a String - how might you define this?)
      • An array of AssignmentGrades (we defined that struct in the lecture)
      • An averageGrade for the student. This property should be a computed property that derives its value from the assignment grades. As part of the exercise, place this computed property in an extension of the Student struct. HINT: You can use functional programming to help compute the average grade!
    • Add the student to the ClassAssignmentsModel struct already defined in the lecture
    • Make a struct that populates data structures from the 4 JSON files located in the Resources folder of the project (a stub for this was made in the lecture video), and use that struct to populate your ClassAssignmentModel. This step will demonstrated your ability to read and process JSON files (see the JSONDecoder class for some help!). Generics can be used here to make your job easier!
    • Be sure to handle error cases when doing things like reading in from the JSON files (see "Error Handling" in Apple's Swift Programming Language Guide and the "Handling Errors" module for a refresher)
  • When you have confirmed that the playground runs as intended, you're ready to copy the new code you wrote back over to the main application
    • Make a file for the Student struct and copy the code from the playground over to that file
    • Update the ClassAssignmentsModel struct from the changes you made in the playground

NOTE: You won't be running anything in the simulator this week; however, the playground should run (be able to load the JSON files, and demonstrate the data structures are populated correctly) and the correct code should be in the project; this will get you setup for next week when you will make UI views and use the simulator.

Assignment Submission

  • Zip up the Xcode project folder. This should be the folder that has the .xcodeproj file inside it.
  • Rename the zip to LastName_Homework#.zip
  • Upload the zip to the Programming Assignment 1 areas on Blackboard BEFORE 11:59 PM ET on the day it is due (Sunday of Week 1).
  • Be sure to test your app before submitting, so you know it works as intended!

Assignment Rubrics

In addition to the normal grading criteria stated in the syllabus, you will be graded on:

  • Use the playground to test your implementation before using it in the project
  • Proper implementation of the Student struct, including
    • The averageGrade computed property
    • Implementation of the Program property
    • Implementation of the array of assignment grades
    • Other specified properties
  • Proper implementation of the data structure populated by JSON files
  • Addition of the student to the existing ClassAssignmentsModel struct
