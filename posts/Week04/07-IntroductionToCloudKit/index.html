Overview

There are various reasons why you might want to store your data in the cloud.

  • Cloud storage allows you to share data across devices, which comes in helpful when your app supports iPhone or iPad, for example.
  • Cloud storage also allows for larger sets of data - that data can sit on the cloud instead of your device, and you can store only the data you need locally.
  • Cloud storage is also good for data that doesn’t require a lot of server side logic - but, it can be used for web and server applications.

CloudKit is a framework that connects your app with CloudKit data stores run by Apple. CloudKit consists of a web dashboard and a set of APIs. Note that you do need a paid iOS developer account to setup credentials for CloudKit.

CloudKit has an appealing set of benefits to entice developers to use their service instead of something like Amazon Web Services.

  • First, it has easy setup - all you need to do is toggle a switch under the Capabilities tab for your project, and Xcode does all the handshaking with the CloudKit servers to enable your project.
  • As mentioned, part of CloudKit is a framework to access the data on the CloudKit servers, and best of all the authentication method for the CloudKit server data is something your users are already intimately familiar with - their Apple ID and password.
  • Another benefit of CloudKit is that each user’s data is sandboxed from the other users on the system, and those same credentials used for logging in set the proper permissions on the filesystem to keep the data properly isolated.
  • Finally, the costs for CloudKit is comparable to that of other services, and those services are designed to work intimately with the client side frameworks.

While CloudKit allows for storage of your app’s data on the web, it’s not necessarily a replacement for your own local data objects.

  • It’s designed to transfer data to and from the cloud, meaning that you still need to manage local storage to cache that information (if you wish).
  • Since it is connecting to the web, it obviously needs a network connection to operate properly. This again implies a local cache - probably in the form of a Core Data database - should probably be paired with your CloudKit usage in order for you app to work well under constrained network conditions.
