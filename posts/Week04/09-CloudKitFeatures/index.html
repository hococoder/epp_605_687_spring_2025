Week 4 - Other CloudKit Features

Subscribing to Changes

You may want to keep up to date with the latest data that is in the cloud. Instead of polling the cloud over and over again, which may take up unnecessarily resources in the form of bandwidth and battery power, you can subscribe to the types of changes you want to monitor. For example if a user marks an artist as a favorite, that could setup a subscription to the database to listen for the addition of new albums for that artist. Subscription information is stored in the database, and notifications are passed to the app through Apple’s push notification service.

Don't forget that when setting up the capabilities for CloudKit earlier, Xcode already enabled your app for Push Notifications too!

CloudKit and your Web Server

CloudKit also has APIs that your web server can use to push and pull data to iCloud storage just like any iOS or macOS App. The CloudKit JS library and Web Services API makes this possible.

More with CloudKit Dashboard

In addition to providing access to the Schema of your containers, as well as the data within them, the CloudKit Dashboard also gives you access to access Logs, Telemetry (showing you requests, server latency and more), usage statistics (users, requests per second, and more), and information about external API access from web servers.

Moving from Development to Production

Moving from development to production is easy - click deployment in the pull down of the development section of your container and then select Deploy to Production. When you do this, be sure to verify your production schema. Also be aware that in future deployments you can only add new record types - you can’t remove records from the schema.
