Week 4 - Setting up CloudKit in Your App

Previous: Introduction to CloudKit

Next: Cloud Kit Schemas



Overview

Let’s talk a bit about getting CloudKit enabled, how to setup schemas, and a few unique CloudKit features. As mentioned earlier, enabling CloudKit is pretty easy, and there are 2 steps to do it.

At Creation Time

First, you can choose to support CloudKit from the creation of your app. The screenshot below shows the new project dialog - if both "Use Core Data" and "Host in CloudKit" are checked, Xcode enables what it can in the source code when the project is created.

What exactly does it change in the source code? We'll take a look at that in the "Updating Your App for CloudKit" module later.

Setting up Capabilities

iCloud and Push Notifications

Whether you enabled CloudKit from the start or not, you still have to add the iCloud capability to your project. Under your project’s capabilities tab (under Project File -> Target -> Signing and Capabilities), click the "+Capability" button and search for and selected iCloud (which also adds Push Notifications for you, which is nice - you can get notifications from iCloud to respond to changes!).

When the display updates, toggle CloudKit to on, and Xcode will communicate with the Developer portal as well as the backend CloudKit servers to configure your project for CloudKit use.

Making a Container

At this point, you can make a new iCloud container for your app to use - give it the same name as your app bundle, but with a unique identifier behind it (I will make some containers ahead of time for you since you need admin rights to make those containers). Again, Xcode will communicate with the back end to set everything up for you. You will also be able to see the other containers you have added under your development account. If for whatever reason things don’t happen automatically, you can force Xcode to try again with the “Fix Issues” button in the capabilities pane.

Background Modes

Starting with Xcode 12, you also have to turn on the "Remote Notifications" of the "Background Modes" capability

Note that in your HW 4 starter project, with the exception of the exact iCloud container to use (you'll select the one that matches your JHED), all of these will be setup for you

You'll also notice there is an "CloudKit Dashboard" button - this opens up a web browser to give you a web view of the CloudKit databases you have access to. We'll come back to that in a minute.
