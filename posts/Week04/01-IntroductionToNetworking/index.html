As you are fully aware, the modern day app ecosystem relies heavily on getting data from the web. Many applications are useless without the data that a network connection provides, which can provide up to date information for the app and user to process. iOS has a mature set of API under the URLSession umbrella that abstracts the details of establishing a connection and transmitting data on that connection.

  • The URLSession API contains delegate methods (see the next module) that can handle activities such as authentication and background downloads, which can occur when the app is not running, or the app is suspended. This allows your app to stay responsive to user interactions
  • URLSession supports data, file, ftp, http and https URL schemes.
  • Tasks within a session all share a common session configuration object, which can specify things such as timeout values, caching policies, among others.

Related Classes

Here are a few related classes for URLSession.

  • The URL class contains a URL (naturally)
  • URLSession.shared is a singleton in URLSession that doesn't have a configuration object, so isn't as customizable as sessions you create
  • URLSessionConfiguration lets you specify properties of the session such as whether cellular access is allows, the dominant data type, and more
  • URLRequest objects holds metadata related to the URL request (the actual URL, the request method, etc)
  • URLResponse object holds metadata related to the server’s response to the request you made.

Here's an example of setting up a URL, the shared session, as well as a custom configuration from the URLSessionConfiguration.default one

let url = URL(string: "http://159.203.191.136/test.json")
let session = URLSession.shared
let configuration = URLSessionConfiguration.default
configuration.allowsCellularAccess = true
configuration.networkServiceType = .video
let defaultSession = URLSession(configuration: configuration)

There are different types of tasks that can run under URLSession.

  • Data tasks can send and receive Data objects, and are typically short, interactive requests.
let dataTask = session.dataTask(with: url!) { data, response, error in
   print("Contacted the webservice and got the data, \(data)")
   let json = try! JSONSerialization.jsonObject(with: data!)
   print("json is \(json), response is \(response)")
   //only have data here, still need to save it....
}
dataTask.resume()

Note that when working with tasks such as these, they are initialized in a "suspended" state. Therefore when you are finished defining them and are ready to kick off the network call, you must call resume() on the task.

  • Upload tasks are similar to data tasks but also send data, often in the form of files, and support background uploads.
  • Download tasks retrieve files, and support background downloads AND uploads while the app is not running.
let downloadTask = session.downloadTask(with: url!) { url, response, error in
   print("Data has been store to \(url!), please save it permanently")
   //this saves data to a temporary file, need to make it permanent
}
downloadTask.resume()

Note here that the downloadTask saves the data to a temporary file, and its up to you to put it in a more permanent location (i.e. somewhere in your App's directory structure).
