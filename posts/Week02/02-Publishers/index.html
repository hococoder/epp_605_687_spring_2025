Week 2 - Combine Publishers

Previous: Introduction To Combine

Next: Combine Subscribers



Overview


On one side, you have Publishers, which are associated with 2 associated types:

  • One to describe the data being published, and
  • One that describes a potential failure type when publishing this data.

An associated type gives a placeholder name to a type used as part of a protocol. It's useful when defining generics - you know you're dealing with a type, but you won't know which type until the concrete class is made. This is perfect for this situation - you will have a data type and failure type in the publisher - you just won't know which ones until the publisher is made.

Built in Publishers

There are 2 built in basic publishers:

  • "Just" for publishing simple values, and
  • "Future" for dealing with data generated by functions that may be asynchronous.

Other frameworks have publishers in them - NotificationCenter and URLSessions for example - that can be used in your code.

import Combine

let publisher = NotificationCenter.default
                                .publisher(for: Notification.Name("edu.jhu.ep.gradebook.gradeschanged")

This code establishes a publisher using the default NotificationCenter's publisher method. This will fire off a notification when someone posts to that Notification.Name type (in particular the one named edu.jhu.ep.gradebook.gradeschanged).

An important thing to remember is that publishers don't send data unless they have a subscriber! We'll talk about those next.
