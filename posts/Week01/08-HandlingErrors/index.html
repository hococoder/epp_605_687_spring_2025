Throwing Errors

Page Contents

  • Throwing Errors
  • Catching Errors

As with Java, functions can throw errors in Swift. You can represent an error condition by adopting the Error protocol:

enum MyError: Error {
  case StupidTypo
  case SemanticMisunderstanding
  case Unknown
}

To indicate that a function can throw an error you use the throws keyword in the function signature:

func check(name: String) throws -> Bool {
  if !names.contains(name) {
    throw MyError.StupidTypo
  }
  return true
}

Catching Errors

Also similar to Java, when a function can throw an error, you need to handle it. In Swift, this is done with the do/catch syntax. For our example above, the code would look like this:

let names = ["Josh"]

do {
  try check(name:"Jash")
} catch {
  print(error)
}

The code that may throw an exception is preceded by a try keyword.
