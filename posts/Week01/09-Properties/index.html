Both Structs and Classes support properties - constants (declared with let) or variables (declared with var) that are stored as part of the type. Initial values can be defined at declaration time or inside an initializer, even for constants:

struct FrequentFlyerPass 
{
  let name: String
  var numberOfMiles: Int = 0
  
  init(name: String) {
    self.name = name
  }
}

Important note: if you declare an instance of the above struct and assign it to a constant, you can't change the properties of the struct, even if that property is a variable. This is because structs are value types - when declared constant, the properties of the value type become constant. This isn't true for Classes.

The above properties are called "Stored" properties, because they are declared either at initialization or declaration time.

Lazy Properties

You may have a property that you don't want to initialize right away - for example, something that might kick off a long running process like making contact with a remote database or reading data from a large file. Properties like this can be given the lazy tag so they are initialized on their first use, not at instance creation.

Computed Properties

Classes, structs and enumerations can also define computed properties, which don't store a value but instead have a getter and setter to retrieve and set other properties and values indirectly. A great example to represent this comes from the Swift Programming Language guide, so I'll put it here:

struct Point {
  var x = 0.0, y = 0.0
}
struct Size {
  var width = 0.0, height = 0.0
}
struct Rect {
  var origin = Point()
  var size = Size()
  var center: Point {
      get {
          let centerX = origin.x + (size.width / 2)
          let centerY = origin.y + (size.height / 2)
          return Point(x: centerX, y: centerY)
      }
      set(newCenter) {
          origin.x = newCenter.x - (size.width / 2)
          origin.y = newCenter.y - (size.height / 2)
      }
  }
}

The center property is a computed property, as you can see by the getter and setter method. For the getter, the values of other properties - the origin and size are used to calculate the center Point. Inside the setter, a new center Point is used to generate new values for the origin based on the newly passed in center (in newCenter) and the existing size.

Property Observers

You can keep an eye on properties to see if they are about to change or just have changed, and respond accordingly. This can be done with willSet and didSet property observers.

willSet is called just before the values is stored, and that new value is passed to the willSet method as a constant

didSet is called immediately after the new value is stored, the old value is passed into the didSet method. That old value can literally be referred to as oldValue, or you can name the parameter:

struct Person {
  var name: String
  let dob: Date
  var age: Int = 0 {
  
    willSet(newAge) {
      print("Getting ready to set the new age to \(newAge)")
    }
    
    didSet {
      print("Set the new age, it was \(oldValue) previously")
    }
  }
}

Property Wrappers

A recent addition (as of Swift 5.1) to properties is the concept of a Property Wrapper. This addition to the Swift API allows you to wrap a property in another type. Put another way, the property wrapper connects the value of the property with an underlying storage type.

This underlying type can not only store values, but automatically apply functions to those values. An example might be a property wrapper that handles the writing of values to a database, or automatically converting a value to a particular unit (for example, taking a Fahrenheit temperature input by the user and converting it to be in Celsius, which is used by the underlying library).

When we get to discussing SwiftUI, you'll see property wrappers all over the place when keeping track of the view's state. This may not make sense until then, but here's an example of using @State in SwiftUI:

struct InputView: View {
  
  @State var username: String = ""
  
  var body: some View {
    TextField("Username", text: $username)
  }
}

This code here shows that the username property has a @State property wrapper, which store that value as part of the state of the view. We'll talk about what the $username argument in the TextField initializer is when we talk about SwiftUI next week.

For this course, we'll be using built in property wrappers, such as those in SwiftUI, but you can also make your own using the @propertyWrapper keyword before the definition of the underlying structure.

Property wrappers inside function and closure parameters

As of Swift 5.5 you can use them in your function and closure parameters like this:

func save(@Capitalized lastName: String) {
    ...
}

This function saves a last name, but has a property wrapper on it that takes the passed in parameter and capitalizes it if it isn't already capitalized.
