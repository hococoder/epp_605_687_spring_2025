Protocols

Page Contents

In Swift, Protocols are similar to Interfaces in Java. You can declare a Protocol using the protocol keyword, and make structs or classes adopt that protocol like this:

protocol Person {

 init(name: String, age: Int)
 var age: Int { get }
 mutating func changeNameTo(name:String)

}

struct Teacher: Person {
  
  var numberOfYears: Int
  var age: Int
  
  init(name: String, age: Int) {
    self.age = age
    self.numberOfYears = 0
  }

  mutating func changeNameTo(name: String) {
    
  }
}

class Principal: Person {
  
  var age: Int
  
  required init(name: String, age: Int) {
    self.age = age
  }

  func changeNameTo(name: String) {
    
  }
}

A few things to make note of here:

  • The protocol can specify initializers that need to be implemented by the adopting type
  • The protocol can define properties as well as methods, and you can specify whether the compiler should auto-generate getters and setters with the get and set keywords
  • Structure methods can be declared as mutating to state that they modify the structure
  • Both type properties and methods are prefixed with the static keyword
  • If a Protocol is adopted by a Class, it must be declared after any parent type
  • One or more protocols can be adopted by a type
  • Protocols that extend the AnyObject protocol can only be used by Classes
  • To accept objects that adopt multiple Protocols, you can use syntax like Teacher & Parent to state that the type here must adopt both Teacher and Parent protocols.
  • Protocols can declare default implementations for methods by declaring them in a Protocol Extension (see below for more on Extensions)

Extensions

One of my favorite features in Swift is the ability to declare Extensions. These allow you to add new functionality to an existing class, structure, enumeration or protocol type. They are similar to Categories in Objective-C.

Yes, it just needs to exist. That means types for which you don't have access to the source code - such as types in any of the iOS frameworks (say, adding a new method onto the String class that can be used anywhere in your code) or third party libraries. Via extensions you can, among others:

  • Add computed properties
  • Define methods
  • Add initializers
  • Make an existing type conform to a protocol

Making an extension uses the extension keyword, and in addition to the uses above, you can also use extensions to group logical part of your struct or class together, since you can have more than 1 extension for a given type. If we wanted to make the Teacher struct above conform to a protocol, we could do that via an extension:

protocol SchoolStaff {
  func numberOfYearsAtSchool() -> Int
}


extension Teacher: SchoolStaff {

  func numberOfYearsAtSchool() -> Int {
    return numberOfYears
  }
}

In addition to making Teacher conform to SchoolStaff the extension also silos off the code related to SchoolStaff in your file.
