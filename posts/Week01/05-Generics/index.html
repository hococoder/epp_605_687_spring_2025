Generics

Similar to Java, Swift supports generics. Functions, Types, Methods (functions within a Class), Classes, Enumerations and Structures can all handle generic types.

func addValues<ValueType: Numeric>(value1: ValueType, value2: ValueType) -> ValueType {
  value1 + value2
}

addValues(value1: 2.0, value2: 3.0)
addValues(value1: 10, value2: 2)

In addition to allowing developers to make their own generic types and functions (and so on), built in collections in Swift such as Arrays, Dictionaries, and even Optionals use generics, and use syntactical sugar to hide that from you.

var groceries =  ["Milk", "Bread"]  is shorthand for:     var groceries: Array<String> = ["Milk", "String"]
var nameToAge = ["Josh": 45]        is shorthand for:     var nameToAge: Dictionary<String, Int> = ["Josh" : 45]
var spouseName: String?             is shorthand for:     var spouseName: Optional<String>

Restricting Generic Types

You may find it useful or necessary to restrict the available types that can be used by a generic type - for example, only allowing types that adopt certain protocols. In addition to specifying the restriction like we did above (see the <ValueType: Numeric> code that specified the type must adopt Numeric), this can be done by using the where keyword right before the body. For example, this code:

func addValues2<ValueType>(value1: ValueType, value2: ValueType) -> ValueType where ValueType: FixedWidthInteger {
  value1 + value2
}

will only allow types that conform to FixedWidthInteger be acceptable for this function. This means, among other things, that Double could not be used as a type here.
