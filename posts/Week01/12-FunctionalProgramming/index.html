As with most modern day languages, Swift has a set of built in Collection types - Arrays, Sets, Dictionaries, and so on - that allow you to hold data in various ways. These types are all part of what is referring to as the Swift Standard Library, and along with the Collection types is a set of functionality that allow you to operate on the values in those collections. This is accomplished with aspects of Functional Programming, which is a type of programming that deals with immutable state as well as issuing no side effects when functions are called. Swift is not a Functional Programming language, but instead combines aspects of Functional Programming along with the Imperative based programming throughout the rest of the language.

Putting the "Function" in "Functional"

Recall that in the module on Closures, we discussed the fact that in Swift, functions (which are just special types of closures) are treated like first-class citizens in Swift - they can be passed around into other functions, assigned to variables, and so on. Let's look at a few built in functions that the Swift Standard Library has for working with collections.

Map

Map takes in a function and applies that function to each element of the collection, and returns a collection of the same length with those modified values.

let helloInLanguages = ["Hello", "Bonjour", "Gutentag", "Hola", "Ni Hao"]
let personalizedGreeting = helloInLanguages.map({ return "\($0) Josh!"})

personalizedGreeting

The above code outputs the following

Hello, Josh!
Bonjour, Josh!
Gutentag, Josh!
Hola, Josh!
Ni Hao, Josh!

Reduce

Reduce takes 2 parameters - the first is a starting value of a given type (say, an Int), and a function that combines the current value with a value from the collection (so, you could start your baseball game with a score of 0, and for each inning of the game, gets the runs scored, and add it to the current value).

let baseballScoresPerInning = [0, 2, 3, 0, 0, 0, 1, 2, 4]
let finalScore = baseballScoresPerInning.reduce(0, +)       //returns 12

Filter

filter also works on and returns collections, and takes in a function. If executing the function with the parameter of the collection returns true, it adds that value to the returned collection (and as described, it filters elements out of the original collection).

let someNumbers = [1, 2, 4, 12, 9, 10, 0, -1]
let lessThan3 = someNumbers.filter({ $0 < 3 })

lessThan3

The above code returns the array [1, 2, 0, -1]

Thanks to the fact that functions take input, process it, and output it, with no side effects, you can easily send the output of one function to another, thereby chaining functions together. You can also compose functions from other functions, thanks to their first class status and their ability to avoid side effects. For example, if we wanted to find the sum of all runs scored in innings where the number of scored runs were less than 3, we could combine the 2 operations above:

let totalRunsScoredInInningsLessThan3Runs = baseballScoresPerInning.filter({ $0 < 3 }).reduce(0, +)

totalRunsScoredInInningsLessThan3Runs   //this returns 5
