Overview

This video covers all of the presentation content for SpriteKit and some of the content is repeated in the subsequent modules, presented before the videos for each section. `SpriteKit` is a rendering and animation library for 2D Graphics. The engine’s rendering loop determines the content of the frame and displays it on screen. `SpriteKit` provides an optimized rendering of the scene using the system hardware, so you don’t have to. But wait! What about INSERT MY FAVORITE LIBRARY HERE! Yes, there are other 2D graphics libraries out there. `SpriteKit` is Apple’s 2D Graphics library though - so we’ll discuss that here. But wait! What about 3D?? `SpriteKit` covers 2D graphics only. `SceneKit` is Apple’s 3D framework on iOS and macOS. Also, with the introduction of iOS 11, both `SpriteKit` and `SceneKit` are playing heavily into what you can do with `ARKit`, Apple’s augmented reality framework. ## Sprites So, what is a Sprite? Let’s a look at a few definitions. Google says that a sprite is “a computer graphics that may be moved on screen and otherwise manipulated as a single entity”. Apple defines a sprite as an “arbitrarily textured image”. `SpriteKit` takes the idea of a sprite a bit further than both of those definitions and introduces effects like filters, masking, and even an integrated physics library. Sprites can be defined by simply providing a PNG file. Then, you can manipulate properties of the sprite such as the x and y scale, and the sprite’s position in the scene. Finally, you can add it to the scene. (This sort of flow should sound very familiar - views are created, have their properties manipulated, and are added to the parents views just like this). ``` let scene = SKScene() let size = CGSize(width: 400, height: 600) let ship = SKSpriteNode(imageNamed: "enterprise") ship.xScale = 0.5 ship.yScale = 0.5 ship.position = CGPoint(x: size.width*0.1, y: size.height*0.5) scene.addChild(ship) ``` You’re not limited to rectangular sprites in `SpriteKit` - `SKShapeNode` can render a shape defined by a `CoreGraphics` path. As you can see in the example in the slide, a circle is created by defining an arc path, which is then passed into the `SKShapeNode`. ![Non-Rectangular Sprites](../../../week06/Slide13.png#centerResizedSlide) ### Sprite Atlases There may be times where one static representation for your Sprite just won’t do - maybe you want to animate it. Sprite Atlases allow you to define a series of textures that can be iterated over with `SKAction`’s `animateWithTextures` method. ``` var walkingFrames:[SKTexture] = [] var characterAtlas = SKTextureAtlas(named: "char") var character:SKSpriteNode? let numImages = characterAtlas!.textureNames.count for var i:Int=1; i <= numImages; i++ {   let textureName = NSString(format:"char%d", i)     let temp = characterAtlas!.textureNamed(textureName as String)     walkingFrames.append(temp) } let temp: SKTexture = walkingFrames[0] character = SKSpriteNode(texture: temp) //other character setup here let interval:NSTimeInterval = 0.1 character!.runAction(SKAction.repeatActionForever( SKAction.animateWithTextures(walkingFrames,timePerFrame:interval, resize:false, restore:true)), withKey:"walkingInPlaceCharacter") ``` ### Adding Textures Speaking of textures, `SKTextures` can be applied to an `SKSpriteNode` or `SKShapeNode`, particles from a `SKEmitterNode`, or tiles in a `SKTileMapNode`. These objects manage the data and other resources required to render the image. Due to their expensive nature, loading and manipulating textures are usually deferred as long as possible, in order to keep your app operating smoothly. In addition to regular images, normal map textures can be used to simulate 3D lighting on a texture, as seen below - the left image does not have any normals, while the right image does. ![SKTextures](../../../week06/Slide15.png#centerResizedSlide) In the next module, we'll look at how to setup a SpriteKit scene in your app, and how you can use `SKSpriteNodes` in your `SKScene`.
