Overview

The Animations system in SwiftUI allows you to specify the state you want to animate a view to, and the system handles the actual animation itself, abstracting away any complexities for you. This power combined with effect modifiers in SwiftUI allow you to animate not only properties like the view's position or size, but also animate effects such as scaling and rotation.

SwiftUI Effect Modifiers

SwiftUI contains a series of Effect modifiers that come in handle when you want to animate properties of your views. If you go the Library in Xcode, and in the Modifiers tab search for "effect", you'll see them listed.

We'll see how to use these when performing animations later on in the module.

Implicit Animations

For certain view attributes, simply changing the value of the attribute will cause SwiftUI to re-render the view, and animate the view based between the original state and final state, without any additional coding. For example, in this code:

Button("Press to move (implicit animation)") {
  self.buttonPosition = CGPoint(x: 175, y: 55)
}.position(buttonPosition)

The Button is originally placed at the position specified by buttonPosition. The button's action however, is to change that position. When that happens, the rendering engine will animate the location of the button from the original position to the new one specified in the action block.

The animation(_:) Modifier

You can, however, be explicit with your animations by using the animation(_:) modifier. This modifier takes in an Animation type, such as easeIn or easeOut. Since you need something to animate, this modifier is usually paired with an effect modifier, such as scaleEffect

@State var amount: CGFloat = 1

Button("Press to move (using animation)") {
  self.amount += 1
}
.scaleEffect(amount)
.animation(.easeIn(duration: 2.0))

The button press changes the amount property, which changes the scaleEffect on the view - to animate this, the system uses the specified animation modifier instead of doing it implicitly, allowing us to specify the style and duration of the animation we want to use.

Using withAnimation(_:)

An alternative way of specifying an animation - or more precisely the things you want to animate - is to wrap them in a withAnimation block. Here's the first example above reimagined with a withAnimation block.

Button("Press to move (explicit)") {
  withAnimation(.easeInOut(duration: 3.0)){
    self.buttonPosition2 = CGPoint(x: 175, y: 75)
  }
}.position(buttonPosition2)

Again, instead of using the implicit animation the system may choose, the withAnimation block specifies the type of animation, as well as the values you want to animate to.

Animating Multiple Effects

You aren't restricted to just animating one effect on a view. Any effects that change - typically due to some @State property changing - get updated at the same time, allowing you to make complex animations that all happen together.

Rectangle()
.scaleEffect(2.0 - animationAmount)
.opacity(Double(2 - animationAmount))
.onTapGesture {
    self.animationAmount = 2
  }
}

This code changes both the scale effect and opacity of the rectangle when the rectangle is tapped. You'll see how this looks in the demo video.

Transitioning Views

SwiftUI also has a transition modifier that allows you to add views into the currently visible view in a number of different ways. For example:

Text("Status Area.")
  .transition(.move(edge: .bottom))

When this view gets added to the existing scene, it slides in from the bottom, instead of just appearing. The animation is handled by the SwiftUI rendering system for you.
