Overview

Great WWDC 2019 Video on SwiftUI Accessibility Accessibility on iOS at developer.apple.com

In this module we’ll discuss features of iOS that allow your app to be accessible, which means usable by people with various special needs.

Why Accessibility?

First of all, why make your app accessible?

  • First, it expands your user base - you’re able to reach people you couldn’t before.
  • Depending on the features you use, someone could use your app without seeing the screen.
  • Most importantly it will help your app meet accessibility guidelines, which will help it get support from various groups that help people with disabilities.

iOS Has Accessibility Built In

How does iOS help with accessibility?

  • You can add captioning and audio descriptions to videos
  • The user can customize the display (for example, larger fonts)
  • The text reader can read parts of the screen
  • Voiceover can read elements of the screen to you
  • Guided Access can simplify the interface between the user and the system, allowing them to stay on task.

Built in Xcode Tools

Xcode and iOS have a number of tools that can help make your app accessible.

  • Interface Builder (used with UIKit) allows you to define accessibility information in your XIB files. This can include labels, hints, traits and identifiers.
  • If you are using macOS Monterey and Xcode 13, a new Accessibility Inspector pane in Xcode gives you a quick glance at your SwiftUI accessibility properties.
  • The Accessibility Inspector tool (a separate tool found in the command line tools for Xcode )can display accessibility information in the simulator.
  • Voiceover can be used to test accessibility - how does your app sound to someone who can’t see the screen well?
  • You can even declare that your custom views adopt UIAccessibility based protocols that allow them to report various elements of accessibility that they support.

We'll look at the Accessibility Inspector in the next module's example.

SwiftUI Support

SwiftUI has a lot of built in accessibility support:

  • Accessibility elements are automatically generated for both built in and custom views, and those views automatically get labels and actions.
  • SwiftUI's declarative framework also allows the Accessibility system to know the order the elements are laid out on screen, allowing the screen reader to list them in the proper order.
  • Some SwiftUI elements, like Image, come with additional initializers that can take in informative labels, giving the VoiceOver reader something to say other than the type.
